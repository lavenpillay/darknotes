DarkNotes. Documentation

A canvas-based note-taking application with workspaces, groups, connections, and full MCP integration for AI-assisted note management.

Overview

DarkNotes is a single-file, offline-capable notes app that runs entirely in the browser. Notes live on a freeform canvas — you can position, resize, connect, and group them freely. When the DarkNotes MCP server is running, your notes become accessible to Claude and other AI tools.

📝 Rich note editing

Mixed text and checklist lines, heading fonts, colors, labels, and word wrap.

🗂️ Workspaces

Unlimited named workspaces, each with its own independent canvas and notes.

🔗 Visual connections

Draw bezier curve lines between notes to show relationships and ideas.

📦 Groups

Cluster related notes inside named, resizable, collapsible group containers.

🗺️ Minimap

Always-visible overview of the entire canvas with a draggable viewport window.

🤖 MCP server

Full CRUD access to your notes from Claude and any MCP-compatible AI client.

Interface

The app has three main areas:

  • Toolbar — fixed top bar with workspace switcher, note count, MCP status indicator, zoom controls, and action buttons.
  • Canvas — the infinite scrollable/zoomable board where notes and groups live.
  • Minimap — collapsible overview panel in the bottom-right corner.

The small coloured dot next to the note count in the toolbar shows the MCP server status: green = online, grey = offline.

Creating Notes

Click the + New Note button in the toolbar. New notes always appear at the centre of the current visible canvas area, accounting for your pan position and zoom level.

You can also duplicate any note by right-clicking it and choosing Duplicate. The copy appears offset 30px from the original.

Editing Content

Each note has a heading (top) and a line editor (body). The body is a line-based editor where every line is independent.

Line navigation

All lines support automatic word wrap — content wraps to the note width and the line expands vertically as you type.

Checklists

Checklists are inline with text — any line can be a plain text line or a checkbox line. You can freely mix them.

Creating checklist items

  • Click the ☑ Toggle Checklist button in the note's hover toolbar to convert the current line to a checkbox (or back to text).
  • Press Tab on any line to toggle between text and checkbox.
  • Press Enter on a checkbox line to create the next checkbox below.
Tip: A note can have any mix of text and checkbox lines in any order — for example, a paragraph, then a checklist, then more text, then another checklist.

Styling Notes

Colors

Hover a note to reveal its toolbar. Click the palette icon to open the color panel, which has 20 background colors and 10 font colors.

Font size

Use the font size dropdown (11–24px) in the note toolbar to adjust the body text size. The heading size is controlled separately in Settings.

Heading font & size

Click the sliders icon (Settings) in the note toolbar to open the heading settings panel. Choose from 19 Google Fonts across 5 categories (Serif, Sans, Mono, Display, Script) and adjust the heading size with a slider (12–36px). Each font is previewed in its own typeface.

Labels

Labels are hashtag-style tags attached to notes. Type a label name in the label input at the bottom of a note and press Enter to add it. Click the × on any label to remove it.

Labels are searchable and filterable via the MCP API.

Arranging Notes

Moving notes

Drag a note by its header bar or any empty area of its body. Notes snap to the top of the z-stack when clicked.

Resizing notes

  • Corner handle (bottom-right) — drag to resize width and height simultaneously.
  • Right edge strip — drag to resize width only.
  • Bottom edge strip — drag to resize height only.

Minimum note size is 220×140px.

Connections

Notes can be visually connected with bezier curve lines to show relationships.

1

Hover a note edge

Orange connection dots appear on the top, right, bottom, and left edges.

2

Drag from a dot

A live preview line appears as you drag across the canvas.

3

Drop on another dot

Release over any connection dot on another note to create the connection.

Connection lines follow notes as they are dragged or resized. Click any connection line to delete it.

Groups

Groups are named, semi-transparent rounded rectangles that sit behind notes, letting you cluster related notes visually.

Creating groups

Click the New Group button in the toolbar. The group appears on the canvas and can be named, repositioned, resized, and colored.

Adding notes to a group

Right-click any ungrouped note and choose Add to Group. A submenu lists all available groups by color and name. The note moves inside the group's bounds if it is currently outside.

Removing notes from a group

Right-click a grouped note and choose Remove from Group. The note stays at its current position on the canvas, now ungrouped.

Minimizing groups

Click the chevron ∨ button in a group's header bar to minimize it. The group collapses to a slim pill showing only its name, and all notes inside are hidden (along with their connection lines). Click the pill or the ∧ chevron to restore everything.

Group controls

  • Drag the header bar to move the group.
  • Resize handle (bottom-right corner) to resize.
  • Color dot — click to open the group color picker.
  • × button — delete the group (notes inside are ungrouped, not deleted).

Navigation & Zoom

Panning

  • Hold Space and drag to pan.
  • Middle-mouse drag to pan.
  • Scroll (without Ctrl) to pan.

Zooming

  • Ctrl+Scroll — zoom toward the cursor.
  • Ctrl+= / Ctrl++ — zoom in.
  • Ctrl+- — zoom out.
  • Ctrl+0 — reset zoom to 100%.
  • Toolbar − / + buttons and the ⊙ reset button.

Zoom range: 25%–300%. Zoom level is saved per workspace.

Minimap

The minimap (bottom-right corner) shows a live scaled-down view of the entire 4000×3000 canvas. Notes are rendered with their actual background colors. An orange rectangle shows the current viewport.

Minimap interactions

  • Drag the orange rectangle to pan the main canvas.
  • Click anywhere on the minimap to jump the view to that location.
  • ∨ / ∧ chevron in the minimap header to collapse it to a slim bar or expand it back.

Workspaces

Workspaces are completely independent canvases. Each has its own notes, groups, connections, pan position, and zoom level.

Workspace switcher

The workspace switcher pill in the toolbar shows the current workspace name. Click it to open the dropdown, which lists all workspaces with their note counts.

Creating workspaces

Type a name in the New workspace name… field at the bottom of the dropdown and press Enter or click +. A starter note is created automatically.

Renaming workspaces

Click the gear icon next to the workspace pill to open the Workspace Settings dialog. Edit the name and press Enter.

Deleting workspaces

Hover a workspace in the dropdown to reveal its × delete button. The last workspace cannot be deleted.

Import & Export

Export — Click the download icon in the toolbar. The current workspace is exported as a .json file named darknotes-{workspace-name}-{date}.json.

Import — Click the upload icon and select a previously exported JSON file. The notes are imported as a new workspace — your current board is not affected.

Clear All — Deletes all notes and connections in the current workspace only (after confirmation).

MCP Integration — Overview

The DarkNotes MCP server exposes your notes to Claude and any Model Context Protocol-compatible AI client. You can ask Claude to read notes, create new ones, check off tasks, reorganise workspaces, and more — without opening the browser.

The server runs two services:

  • MCP stdio server — Claude Desktop and Claude Code connect here to call tools.
  • REST API (port 3737) — the browser app syncs here so Claude always sees fresh data.
Note: All data is stored in ~/.darknotes/data.json. The app works fully offline without the server — localStorage is always kept in sync as a fallback.

Setup

1

Install dependencies

cd darknotes-mcp
npm install
2

Start the server

# API + MCP together (recommended)
node src/server.js --both

# API only (browser sync, no MCP)
node src/server.js --api

# MCP only (Claude Desktop / Claude Code)
node src/server.js
3

Configure Claude Desktop

Add to claude_desktop_config.json:

{
  "mcpServers": {
    "darknotes": {
      "command": "node",
      "args": ["/absolute/path/to/darknotes-mcp/src/server.js"]
    }
  }
}

Config location — macOS: ~/Library/Application Support/Claude/claude_desktop_config.json
Windows: %APPDATA%\Claude\claude_desktop_config.json

4

Configure Claude Code

claude mcp add darknotes node /absolute/path/to/darknotes-mcp/src/server.js
5

Verify connection

Open notes-app.html. The dot next to the note count turns green when the API is reachable.

Usage Examples with Claude

Once the MCP server is configured, you can talk to Claude naturally:

Reading notes

"Show me all my notes in the Work workspace"
"Search my notes for anything about the Q3 report"
"List all notes tagged #urgent"
"What are the incomplete items in my Shopping checklist note?"

Creating content

"Create a note called 'Meeting Notes' with today's agenda items as a checklist"
"Add a new workspace called 'Side Project' and create three notes for tasks, ideas, and research"
"Append action items from our conversation to my 'Project Tracker' note"

Organising

"Create a group called 'In Progress' and move all notes tagged #active into it"
"Connect the 'Requirements' note to the 'Design' note on the left side"
"Check off the first two items in my grocery list note"
"Move all notes mentioning 'marketing' to a new workspace"

Data & Privacy

All DarkNotes data is stored locally on your machine:

  • Browser: localStorage (under the darknotes_* keys) keeps the app working offline at all times.
  • MCP server: ~/.darknotes/data.json — plain, human-readable JSON. Back it up, version-control it, or copy it between machines freely.

No data is sent to any cloud service. The MCP server only communicates locally — via stdio to Claude, and HTTP on localhost.

Backup tip: Copy ~/.darknotes/data.json to back up all your workspaces, or export individual workspaces from the toolbar for sharing.
